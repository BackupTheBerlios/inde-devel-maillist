Okay, just to explain the added functions:

dirSeparator():
    should be clear

hiddenName(FXString&):
    On Linux a hidden file is identified by the leading "." in the 
filename. On Windows this is unimportant. This
    function returns either ".filename" or "filename", dependent on 
which platform you are on.

configFilename(FXstring&):
    On Windows, filename extension are useful for associating programs 
with a filetype. On Linux this is mostly
    obsolete due to mime-types (yes?). So this function returns 
"filename.ini" on Windows but "filename" on Linux.

That's all. Those functions at least removed #ifdef WIN32 out of the 
MainWindow constructor. Hehe.
