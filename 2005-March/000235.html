Yes, I had that in mind. To be honest, I've already written it. Hehe.

An iterator class for files in the project would be great. I don't think 
that's oversized.
This way the project meta class would have to wrap the files listed in 
project.inde into
a simple struct/class structure. Afterwards it would be easy to write an 
iterator for that.

That's just an idea but could be useful for the project browser for example.

Besides I'm somewhat thinking of separating InDE into the program itself 
and
a "InDE Foundation Library". This means we would add all these classes 
we need
into a library and then use them in InDE itself. The program's code 
itself was much
shorter, then - and other developers could use the library to build 
editors etc.

That would too much for the first release, but thereafter ... - what do 
you think?
