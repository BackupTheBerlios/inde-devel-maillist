I have some ideas, but haven't thought in depth to all upcomming
problems. This will most probably be a topic next week(end).

But on the other hand, we could arrange this to be solved in later
versions more intelligent. As far as we use the project settings, one
can specify all needed libraries there and depending on this the correct
SConstruct is build. As soon as you move the project to another
plattform, it is needed to revise the project settings to change the
libraries.
I guess, that will do it for the very first release. (I also guess that
noone will use InDE 0.1 for large projects and especially not for too
complicated cross plattform projects)

All in depth SConstruct buildings depend on the task i added about the
project settings to know the compiler and OS. 

For all this we need to know compiler switches, library names (or
differences in them), linker options and so on. This will also be very
interesting for debugging support.....
