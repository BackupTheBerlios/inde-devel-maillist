I forgot to mention that I'm writing classes for the languages itself at 
the moment.

Those derive from an abstract class "Language" and have mostly static 
methods for
validation the names of types, parameters, functions, classes and 
modifiers as well
as some methods for retrieving the default file extension for that 
language and so on
and so on ...

This means: Every programming language we want to support require two 
classes
to be written: <Insert Language alias here>ClassBuilder and
<Insert Language alias here>Language. Examples would be:
    - CppClassBuilder, CppLanguage
    - RubyClassBuilder, RubyLanguage
    - JavaClassBuilder, JavaLanguage

Just to make my last mail complete.
