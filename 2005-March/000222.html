Hi again.

Some thoughts on ...

Modularization:

I just browsed the Mozilla source code today and learned XUL + XPCOM for 
about ten hours.
While doing so I noticed that it is of great importance to keep your 
code as modular as
possible. Thus, I'm planning to separate the styling part out of 
TabEditor into a new class
with a name like StyleEngine. That way we could - just for example - 
provide syntaxhighlighted
editor widgets anywhere without having to use the TabEditor. Might be 
useful for editing
the syntax file some day or if you want to detach the current document 
from the TabEditor to
a single-document window. That's basically just ideas.


Coding style:

Perhaps we should define some code styling conventions. This would make 
the code more
readable for everyone. Personally, I like the code style they're using 
at Mozilla (w/o emacs line):
    http://www.mozilla.org/hacking/mozilla-style-guide.html#Visual
I'd prefer to use it instead of the mixed code we're using at the moment.


That's it for now. What do you think?

- Jannis
