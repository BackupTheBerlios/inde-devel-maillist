Am Dienstag, den 08.03.2005, 20:28 +0100 schrieb Jannis Pohlmann:
> Hi Jochen.
> 
> As already mentioned, I'd like to use NaturalDocs (naturaldocs.org) in 
> order to generate the API documentation
> for InDE.
> 
> I've commited my latest work to SVN and added a file called
>    
>     gendoc.sh
> 
> You can download NaturalDocs, and (for example) move it to
>    
>     /opt/NaturalDocs/
> 
> If you then create a symlink like this:
> 
>     ln -s /opt/NaturalDocs/NaturalDocs /usr/local/bin/NaturalDocs
> 
> You can simply run ./gendoc.sh in order to generate the documentation. 
> Since this is absolutely optional, I
> don't think we should integrate it into the SConstruct file for InDE.
> 
We could add a switch for this. Maybe something like:
	
	scons -GEN_DOCS

to automatically build the docs.

What about a cvn tree for the api docs. As far as i remember, there is a
possibility on berlios to generate webcontents from svn trees. So we
just have to commit the api-doc, but do not have to update the website
-- this job would be done by berlios. 
BTW, they provide nice scripts for the svn tree's, e.g. for a svn
Mailinglist: the script generates a mail to notify the users that new
versions have been commited to svn,....
